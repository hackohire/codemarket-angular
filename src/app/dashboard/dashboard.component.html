4/14/2020 Scrumbmaster: a tool for agile / RAD rapid application development.
Traditional development life cycle from a business person gives requirment to designer, frontend developer, backend developer, deploy takes a few monthes. If one can immediatly post to give feedback, the project don't have to waste money on waiting time for a month.
what's methology is a develpment methology. It's a project managemet style.
The best way for a product manager to do this is to highlight what's not working through communication in a collobration plateform.
Scrumb meeting: everyday standard meeting (everyone: business person, developer, designer, project manager for 10 minutes) or posting what you do. Scrumb meeting forces everyone to talk. vs. e-mails: not good, because people don't read all the e-mails; and some of the e-mails could be deleted. Ideal: e-mail + tools to make people on same page. Agile development: Flexible. Requirement is not frozen; and development is not frozen. Requirement can be kept changing.
Scrumb manager = kind of like project manager Scrumb = everyone doing certain things Scrumb master = manage certain things
